Unigrid Style Guide

Unigrid is a React component that implements a configurable table composed of lower-level multipurpose reusable components.

Unigrid

Uses tag <table>. The main component, contains the following components:

UnigridHeader (optional)
Semantically it's the main header of the whole table.
UnigridSegment (zero or more)
Semantically it's the body of the table, all the rows between UnigridHeader and UnigridFooter. A simple table will contain exactly one UnigridSegment. A "multitable" (tables within tables) will contain multiple UnigridSegment components, one for each nested table.
UnigridFooter (optional)
Semantically it's the main footer of the whole table.

More complex tables, e.g. containing multiple headers or footers, or more than one level of nested tables, should be implemented by splitting the complex table into multiple separate Unigrid components. Those components should then be placed on a page within a new dedicated component built using non-table tags (i.e. <div> or similar that are semantically correct depending on the context within the page). Unigrid components can not be nested (can not be contained within other Unigrid components).

UnigridHeader

Uses tag <thead>. Contains zero or more UnigridRow components.

UnigridSegment

Uses tag <tbody>. Contains zero or more UnigridRow components, each of which can contain either UnigridHeaderCell or UnigridCell components.

UnigridFooter

Uses tag <tfoot>. Contains zero or more UnigridRow components.

UnigridRow

Uses tag <tr>. Contains zero or more of either UnigridHeaderCell or UnigridCell (or a mix of them). It's the basic component to build rows of the table.

UnigridHeaderCell

Uses tag <th>. This is the parent component for all types of cell components that can be shown in the table or segment headers.

UnigridCell

Uses tag <td>. This is the parent component for all types of cell components that can be shown in rows of table segments.
